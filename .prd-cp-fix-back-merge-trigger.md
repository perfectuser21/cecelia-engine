## PRD - 修复 back-merge workflow 误触发问题

**分支**: cp-fix-back-merge-trigger
**需求来源**: CI 调查报告发现 back-merge workflow 在所有分支（develop、cp-*等）都会运行并失败

**功能描述**:
修复 `.github/workflows/back-merge-main-to-develop.yml` 的触发问题。该 workflow 配置为只在 push 到 main 分支时运行，但由于 GitHub Actions 的行为特性，当 workflow 文件本身被修改时，会在所有分支尝试运行（即使触发条件不匹配），导致大量失败记录。

添加 workflow 级别的 `if` 条件，在 job 开始前就阻止在非 main 分支运行，避免无意义的失败记录。

**涉及文件**:
- `.github/workflows/back-merge-main-to-develop.yml` - 添加 `if: github.ref == 'refs/heads/main'` 条件到 `check-trigger` job

**成功标准**:
1. workflow 文件添加了 `if` 条件
2. 在非 main 分支 push 时，workflow 完全不运行（而非运行后 skip）
3. 在 main 分支 push 时，workflow 正常运行
4. CI 通过，无新增失败

**非目标**:
- 不修改 workflow 的其他逻辑（创建 PR、auto-merge 等）
- 不改变触发事件配置（仍然是 `push: branches: [main]`）
- 不影响 workflow 在 main 分支的正常功能
