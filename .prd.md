# PRD - v9.1.1 RCI Baseline Coverage Checkpoint

Priority: P1

## 目标

全量对齐扫描 + 基线快照，证明业务入口被 RCI 覆盖。

## 背景

v9.1.0 完成了 GCI 草稿自动生成，但缺少"我知道现在 OK 了"的节点。
需要一个 Baseline Checkpoint 来证明：当前业务能力都被 RCI 覆盖。

## 价值

- 一次性收口：在 v9.1.1 这个点上生成基线快照
- 覆盖率证明：不再靠感觉，有数据说话
- 增量维护：从此以后只做增量变化

## 业务入口定义（zenithjoy-engine）

| 类型 | 路径模式 | 说明 |
|------|----------|------|
| Skills | `skills/*/SKILL.md` | 用户可调用的 Skill |
| Hooks | `hooks/*.sh` | Claude Code Hooks |
| Scripts | `scripts/*.sh` | 顶层可执行脚本 |
| DevGate | `scripts/devgate/*.cjs` | DevGate 工具脚本 |

## 交付物

1. `scripts/devgate/scan-rci-coverage.cjs` - 扫描器
2. `artifacts/baseline-coverage.json` - 覆盖率数据
3. `artifacts/BASELINE-SNAPSHOT.md` - 人类可读快照

## CI 集成

- `npm run coverage:rci` - 检查 RCI 覆盖率
- 未覆盖 > 0 时警告（暂不阻塞，后续可升级为阻塞）

## 非目标

- 不修改 RCI 结构
- 不添加新的 RCI 条目（本次只做扫描）

## 成功标准

- 扫描器能正确枚举业务入口
- 能正确解析 RCI 并计算覆盖率
- 生成的基线快照包含完整信息
