# PRD - v9.1.3 RCI 覆盖率验证强化

Priority: P1

## 背景

v9.1.2 实现了 RCI 100% 覆盖率，但存在可信度问题：
1. 没有 `--explain` 模式输出审计证据
2. CI 没有集成 `coverage:rci` 检查，无法阻止未覆盖入口合并

## 目标

让 100% 覆盖率成为"可验证的真实"，而不是"自嗨数字"。

## 验证需求

| 验证项 | 当前状态 | 目标 |
|--------|----------|------|
| 分母验证（入口来源） | 无审计输出 | --explain 显示 |
| 分子验证（匹配证据） | 无详情 | --explain 显示 |
| CI 守门 | 未集成 | DevGate 阻塞 |
| 反证测试 | 本地通过 | CI 也能拦截 |

## 交付物

1. `--explain` 模式：输出分母来源和分子匹配证据
2. CI 守门：`coverage:rci` 集成到 DevGate 检查

## 成功标准

- `npm run coverage:rci -- --explain` 输出详细审计报告
- CI 中 DevGate 检查包含 RCI 覆盖率
- 反证测试：新增未覆盖入口会导致 CI 失败
