# PRD: 允许 release-* 分支创建发布 PR

## Priority
P0

## 背景
当前 pr-gate-v2.sh 的 Release 模式只允许从 develop 分支创建 PR 到 main。但当 develop 和 main 有冲突时，无法在 GitHub Web 上解决（develop 有分支保护），需要使用 release-* 分支解决冲突后再创建 PR。

## 目标
修改 pr-gate-v2.sh，允许 release-* 分支在 Release 模式下创建 PR 到 main。

## 需求
1. pr-gate-v2.sh 的分支检查逻辑：
   - PR 模式：仍然只允许 cp-* 和 feature/* 分支
   - Release 模式：允许 develop 或 release-* 分支

2. 不影响现有的其他检查逻辑

## 成功标准
- 从 release-* 分支可以创建 PR 到 main
- 从 cp-* 分支仍然不能创建 PR 到 main（在非 release 模式）
- Release 模式的所有其他检查仍然生效
