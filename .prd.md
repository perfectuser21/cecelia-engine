# PRD - v9.1 Auto-Draft Gate Contract

Priority: P1

## 目标

当 PR 改到 Gate 相关文件时，AI 自动生成 GCI 契约草稿，用户只需确认。

## 背景

v9.0.0 建立了 RADNA 体系：
- L0 Rules（宪法）
- L1 Contracts（GCI + RCI）
- L2 Executors
- L3 Evidence

现在卡住的是：契约还需要"手写"。

## 价值

- 从"执行者"变成"审批者"
- 减少 50% 心力开销
- Gate 契约永不漏更新
- 体系真正"自我维持"

## 触发条件

改到以下文件时自动触发：
- `hooks/*`
- `scripts/run-gate-tests.sh`
- `scripts/devgate/*`
- `tests/gate/*`
- `.github/workflows/ci.yml`
- `contracts/gate-contract.yaml`

## 输出格式

```yaml
- id: Gx-xxx
  name: "简短描述"
  priority: P0|P1|P2
  trigger: [PR|Release]
  scenario:
    given: "前置条件"
    when: "触发动作"
    then: "预期结果"
  reason: "为什么需要这个契约"
```

## 集成点

- `/assurance` Skill 调用此功能
- PR 提交前自动运行
- 生成草稿到 `artifacts/gci-draft.yaml`

## 非目标

- 不自动修改 `contracts/gate-contract.yaml`（需人工确认）
- 不处理 RCI（留给 v9.2）
