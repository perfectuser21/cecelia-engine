# PRD - 修复 PR Gate 检查目标仓库

**需求来源**: 在其他项目目录运行 `gh pr create --repo X` 时，PR Gate 检查的是当前目录而不是目标仓库 X

**功能描述**:
1. 修改 `pr-gate-v2.sh`，解析 `gh pr create` 命令中的 `--repo` 参数
2. 如果有 `--repo` 参数，切换到对应仓库目录进行检查
3. 如果没有 `--repo` 参数，保持当前行为（检查当前目录）

**成功标准**:
- [ ] 在 zenithjoy-core 目录运行 `gh pr create --repo zenithjoy-engine` 时，检查 zenithjoy-engine
- [ ] 没有 `--repo` 参数时，保持原有行为
- [ ] 解析 `--repo owner/repo` 格式正确
